\section{Conclusion}

The LightsPlease architecture delivers a cloud-native, managed-services-first design for a consumer smart-home platform built on AWS. A loosely coupled serverless backend handles remote device control, automation, telemetry, and camera support in a way that scales with household and device growth while minimizing operational burden for a small team. The primary limitations are the single-region initial deployment, cost sensitivity around camera storage, and dependence on AWS-specific services — all areas that would be addressed as the platform matures and the user base grows.
