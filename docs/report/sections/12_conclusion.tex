\section{Conclusion}

The LightsPlease architecture presents a cloud-native, managed-services-first design for a consumer smart-home platform built on Amazon Web Services. The solution combines remote device control, automation, telemetry handling, and camera support through a loosely coupled serverless backend, allowing the platform to scale as the number of households and connected devices grows. This approach keeps the system aligned with the needs of the kata while avoiding unnecessary operational complexity for a small engineering team.

The primary strengths of the architecture are its scalability, security, and operational simplicity. By relying on managed AWS services, the design supports responsive remote control, durable data storage, and resilient device communication without requiring the team to manage large amounts of underlying infrastructure. These decisions also make the platform easier to extend as new device types, automation features, and user needs emerge over time.

The architecture still carries several important limitations. The most notable are the use of a single AWS region for the initial design, the cost sensitivity of camera storage and video retrieval, and the platform's dependence on AWS-specific services. Future improvements would include a more explicit multi-region resilience strategy, stronger controls around high-volume media costs, and a more fully defined device communication protocol to better support long-term interoperability and growth.
