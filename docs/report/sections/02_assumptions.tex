\section{Assumptions and Constraints}

This design assumes the product is intended for consumer households rather than businesses or multi-site deployments. The platform is expected to support thousands of households within the first three years, with each home using an existing broadband connection and Wi-Fi router for device connectivity. Because the system is meant to be turnkey, onboarding, device registration, and automation setup must remain simple enough for non-technical users.

The software team is responsible for defining the communication protocol used by the modular devices, while hardware design and low-level firmware are handled by other teams. The platform must support secure remote access, user-defined automation, and the addition of new device types without requiring major architectural changes. The platform assumes a default 30-day retention window for detailed telemetry and camera data.

To keep the solution practical for a small operations team, the initial design favors managed AWS services and a single-region deployment, with broader geographic redundancy deferred until the product reaches larger scale. Remote access is assumed to use encrypted communications and strong authentication, including multi-factor authentication. In addition, the cloud platform is designed around a 99.9\% availability target and a normal remote-command latency target of no more than two seconds.
