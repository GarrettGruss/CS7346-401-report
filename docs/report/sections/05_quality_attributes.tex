\section{Cloud Architecture Quality Attributes}

LightsPlease is expected to control household devices over the internet, remain responsive for common actions such as locking doors or changing thermostat settings, and continue operating as the number of homes and connected modules grows. These requirements make security, availability, scalability, performance, fault tolerance, and durability some of the most important quality attributes in the design \cite{cloud_computing}.

These priorities influenced the architecture toward AWS managed services that reduce operational complexity while still supporting a dependable consumer platform. Security needs led to the use of Amazon Cognito and AWS IoT Core so that both users and devices can be authenticated through managed identity mechanisms rather than custom-built access controls. Scalability and cost efficiency pushed the design toward serverless compute and on-demand managed storage, since those services allow the platform to grow with household adoption and device activity without requiring large amounts of fixed infrastructure up front. Availability, reliability, and fault tolerance drove the use of services such AWS Lambda and Step Functions, to provide stateless compute, and retry-based workflow handling. Performance considerations shaped the use of CloudFront and purpose-built managed data services so that common user interactions remain responsive without forcing all traffic through a single backend path. The use of loosely coupled managed services lends itself to extensibility, as it allows for easier integration of new device types and platform capabilities over time without requiring a complete system redesign. 

The main tradeoff of this architecture is that these choices reduce operational burden and improve resilience, but they also increase dependence on AWS-managed services and leave the initial single-region design more exposed to a full regional outage than a multi-region deployment would be.
